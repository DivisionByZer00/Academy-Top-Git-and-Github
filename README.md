## Homework 01

### Тема: "Введение в Git. Основные понятия и установка."

**Задание:**

**Теоретическая часть:**
- Перечислите основные преимущества использования Git.
- Опишите, какие проблемы решает Git при командной разработке.
- Объясните разницу между локальным и удалённым репозиторием.

**Практическая часть**
- Установите Git на вашу операционную систему.
- Настройте глобальные параметры Git (имя пользователя и email).
- Создайте локальный репозиторий в новой папке.
- Создайте текстовый файл с описанием вашего проекта.
- Зафиксируйте изменения в репозитории с комментарием.
- Внесите изменения в созданный файл и создайте новый коммит.
- Просмотрите историю коммитов вашего репозитория.

**Аналитическая часть**    
Опишите, какие проблемы могут возникнуть при работе с Git, если не настроить корректно имя пользователя и email.    
Объясните, почему важно писать осмысленные комментарии к коммитам.    
Представьте ситуацию: вы случайно удалили важный файл в репозитории. Как Git может помочь восстановить его?

**Форма сдачи:**    
Отчёт в текстовом формате (DOCX или PDF), содержащий:
- Ответы на теоретические вопросы
- Скриншоты выполнения практических заданий
- Развёрнутые ответы на аналитические вопросы

## Homework 02

### Тема: "Основы работы с Git. История изменений и отмена действий."

**Теоретическая часть.**    
**Вопросы:**
- Объясните разницу между git reset --soft, --mixed и --hard
- Почему git revert считается безопаснее git reset для командной работы?
- Что происходит с историей коммитов при использовании git revert?

**Практическая часть**

**Задание 1:** Создание истории коммитов
1. Создайте новый репозиторий
2. Создайте файл poem.txt и сделайте 4 коммита, добавляя по одной строфе
стихотворения в каждый коммит
3. Зафиксируйте результат командой git log --oneline –graph

**Задание 2:** Работа с отменой изменений
1. Внесите изменения в последнюю строфу (не добавляя в коммит)
2. Отмените эти изменения с помощью git restore
3. Добавьте новую строфу и проиндексируйте её, затем отмените индексацию

**Задание 3:**  Эксперименты с reset
1. Сделайте новый коммит с изменениями
2. Попробуйте по очереди:
   - git reset --soft HEAD~1
   - git reset --mixed HEAD~1
   - git reset --hard HEAD~1
3. Опишите разницу в результатах

**Задание 4:** Практика с revert
1. Найдите хеш второго коммита через git log
2. Выполните git revert для этого коммита
3. Объясните, какие изменения произошли в файле

**Аналитическая часть**
1. Создайте ситуацию:
   - Сделайте 3 коммита
   - Выполните git reset --hard HEAD~2
   - Восстановите потерянные коммиты через reflog
2. Ответьте на вопросы:
   - Какие команды вы использовали для восстановления?
   - Почему reflog помогает в таких ситуациях?

**Творческое задание.**    
Создайте сценарий, где:
- Два разработчика одновременно работают над файлом
- Один делает reset --hard, другой - revert
- Опишите возможные последствия

**Требования к оформлению:**    
Формат сдачи:
- Текстовый документ с ответами на вопросы
- Скриншоты выполнения команд
- Ссылка на GitHub-репозиторий (если создавали)

**Создание репозитория**
1. На GitHub создайте репозиторий algorithm-practice
2. Локально создайте папку и инициализируйте Git:
   - mkdir algorithm-practice && cd algorithm-practice
   - git init
4. Свяжите с удалённым репозиторием и сделайте первый push

**Ветвление и PR**
1. Создайте ветку sort-algorithms
2. Добавьте файл sort.py с реализацией 2 алгоритмов сортировки
3. Создайте Pull Request на GitHub и назначьте ревьювером преподавателя (укажите его логин)

**Конфликты**
